# Multilingue (FR / EN / ES)

> Annexe de [`CLAUDE.md`](CLAUDE.md). Localisation .NET, colonne `language`, langue au login.

## Textes
Ne **JAMAIS** écrire un texte affiché en dur. Injecter le localizer puis utiliser une clé :
```razor
@inject IStringLocalizer<SharedResource> L
...
@L["Ma.Cle"]
```

## Traductions
Dans `Resources/`, un fichier `.resx` par langue (`SharedResource.resx` = FR par défaut/repli,
`SharedResource.en.resx`, `SharedResource.es.resx`). `SharedResource.cs` est une classe « marqueur »
vide qui sert de point d'ancrage. Configuré dans `Program.cs` :
`AddLocalization(o => o.ResourcesPath = "Resources")` + `UseRequestLocalization(...)` (cultures `fr`/`en`/`es`).

## Changer de langue (sélecteur du header)
Le sélecteur FR/EN/ES appelle l'endpoint **`/Culture/Set?culture=xx&redirectUri=...`** (dans `Program.cs`),
qui pose un **cookie de langue** (`.AspNetCore.Culture`) puis **recharge** la page. Un vrai rechargement
est nécessaire (le rendu serveur relit le cookie au démarrage de la requête).

## Langue au login
L'endpoint `/auth/login` pose le **même** cookie de langue depuis `user.Language` → après connexion,
l'UI bascule sur la langue enregistrée dans le profil.

## Colonne `language`
La langue d'un enregistrement (utilisateur, formation, template d'email…) est **toujours** stockée
dans une colonne nommée **`language`** (valeurs `"fr"` / `"en"` / `"es"`), propriété C# `Language`.
Ne pas utiliser `lang`.
